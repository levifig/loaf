# Accessibility (A11y)

## Target: WCAG 2.1 AA

- Contrast: 4.5:1 normal text, 3:1 large text
- All interactive elements keyboard accessible
- Focus visible on all interactive elements

## Conventions

- Semantic HTML first (`nav`, `main`, `article`, `button`), ARIA second
- Icon buttons: `aria-label` on button, `aria-hidden` on icon
- Forms: `htmlFor`/`id` linking, `aria-describedby` for help/errors, `aria-invalid` on error
- Modals: `role="dialog"`, `aria-modal="true"`, focus trap, Escape to close
- Live regions: `aria-live="polite"` for status, `aria-live="assertive"` for errors
- Skip link: `sr-only` visible on focus
- Focus styles: `focus:ring-2 focus:ring-blue-500 focus:ring-offset-2`

## Testing

- Automated: `jest-axe` with `toHaveNoViolations()` in component tests
- Manual: keyboard navigation, VoiceOver/NVDA, 200% zoom

## Critical Rules

- Use semantic HTML elements (not divs for buttons)
- Never rely on color alone
- Never trap keyboard focus
- Always provide text alternatives

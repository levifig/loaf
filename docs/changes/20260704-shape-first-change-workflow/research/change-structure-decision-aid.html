Q2 · loaf change check failure semantics

Same three scenarios against each option: a valid shaping-stage doc, a banned-frontmatter violation, and an executable doc under the CI gate. The crux is scenario 1 — shaping is a valid state.

A · Two-tier + opt-in gate recommended
Violations always fail; executability is a report; the flag becomes the CI gate on non-draft PRs.

S1 — shaping-stage doc (product only)

$ loaf change check ✓ structure valid — no violations ℹ executable: no — missing Planning Contract, Implementation Units, Verification Contract, Definition of Done exit 0

S2 — banned frontmatter present

$ loaf change check ✗ violation: status-like frontmatter `readiness: shaping` — banned field class (V1) exit 1

S3 — executable doc, CI gate

$ loaf change check --require-executable ✓ structure valid · executable: yes exit 0

CI wiring — the D12 payoff

# draft PR = shaping; ready PR must be executable - run: loaf change check - if: !pull_request.draft run: loaf change check --require-executable
B · Strict by default
Non-executable fails unless --shaping is passed. Inverts the common case.

S1 — shaping-stage doc

$ loaf change check ✗ not executable (4 sections missing) exit 1 $ loaf change check --shaping ✓ structure valid exit 0

S2 — banned frontmatter

$ loaf change check ✗ violation: `readiness:` banned exit 1

S3 — executable doc

$ loaf change check ✓ valid · executable exit 0
Every draft PR runs red until each caller remembers --shaping — hooks, CI, and skills all carry the flag for the majority case.
C · Report-only + separate gate verb
check never fails; a second command owns enforcement.

S1 — shaping-stage doc

$ loaf change check ℹ executable: no (4 missing) exit 0

S2 — banned frontmatter

$ loaf change check ℹ found: `readiness:` frontmatter exit 0 ← violation reported, not enforced $ loaf change gate ✗ `readiness:` banned exit 1

S3 — executable doc

$ loaf change gate --require-executable ✓ pass exit 0
Contradicts V1 as written (check promises exit non-zero on banned frontmatter) — and `gate` is a new verb with no ceremony of its own yet.
A: one command, honest defaults — shaping passes, violations never do, and the flag maps 1:1 onto the draft/ready PR state. The gate the model promised, wired to GitHub.
B: stronger-looking default that punishes the most common state; the flag burden lands on every caller instead of only on CI.
C: cleanest separation on paper, but it breaks V1's existing promise and mints a verb that fails the "name the ceremony" test.

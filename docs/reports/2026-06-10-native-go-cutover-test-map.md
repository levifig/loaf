## Maintenance Rule

When a migration slice adds or changes native behavior, update this map in the same commit as the code and tests. Each new command path should have at least:

- one SQLite-ready behavior test,
- one missing or invalid state test,
- one markdown-only delegation or explicit no-op test when compatibility remains,
- one source-mutation test if the command is intended to be read-only.

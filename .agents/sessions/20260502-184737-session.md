**Key Decisions**

- ADR bar: "architecturally significant" + "difficult to reverse" (Microsoft Well-Architected canonical), not "irreversible". Operationalized as `(Q1 OR Q2) AND Q3` Triage Gate. Q2 = cost of divergence with Now/Later prongs (Now: security regression, contract or interface break, multi-PR coordination; Later: foundational shape commitments whose future reversal cost is the reason to record now).
- ADR ≠ SPEC. ADRs capture *choices* (require credible alternatives considered); SPECs hold development direction with implementation evidence. Principles/manifestos route to `ARCHITECTURE.md` / `VISION.md`. AWS canonical: "separate design from decision."
- ADR Lifecycle: 5 statuses (`Proposed | Accepted | Rejected | Deprecated | Superseded`); original `Decision`/`Context`/`Rationale`/`Consequences` sections immutable; recategorization (deprecate-in-place when the classification was wrong, the rule still holds) distinct from supersession (new ADR replaces old). `## Deprecated`/`## Rejected` body sections required for those statuses; `## Superseded` body section optional (the linkage carries it).
- Frontmatter schema is structured what+when only: `status`, `date`, `accepted_date`, `rejected_date`, `deprecated_date`, `supersedes`, `superseded_by`. Context (`reason`, `migrated_to`) belongs in body section's prose, not duplicated in frontmatter.
- Adversarial Review for substantive guidance changes — `loaf:reviewer` (internal-consistency) is the baseline; `codex:rescue` (adversarial design) is optional/plugin-dependent. Each catches different defect classes.
- Changelog discipline — user-perspective prose only; drop internal/session terms (spec/task IDs, internal Q-numbering, gate-logic shorthand, hook IDs that aren't user-facing).

**Discoveries**

- Canonical-sources synthesis (AWS Prescriptive, Microsoft Well-Architected, Nygard 2011, adr.github.io, MADR, Wikipedia ASR) converges on the bar's exact phrasing.

- Two-reviewer pattern (`loaf:reviewer` + `codex:rescue`) genuinely additive — not redundant. PR #46 had defects each reviewer caught alone.
- ADR-006's first deprecation prose overclaimed "no alternative considered" — Codex caught the factual error (the original Context section did name an alternative); corrected to "alternative named but rejected on philosophical grounds, not architectural."
- ADR frontmatter (`supersedes`/`superseded_by`/`deprecated_*`) has zero CLI consumers today. Schema is forward-looking — frontmatter for indexing + future tooling, body sections are authoritative narrative.
- `loaf release --post-merge` guardrail 3 conflicts with the squash-subjects-reflect-the-work convention (expects `chore: release v<semver>` at HEAD; squash subject is `refactor:`/`chore:` reflecting actual work). Manual fallback used for all three releases.
- Direct push to `main` allowed only for `.agents/` and `docs/`; substantive `content/`/`dist/`/`plugins/` changes require branch + PR.

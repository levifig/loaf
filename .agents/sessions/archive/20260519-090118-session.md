**Prior art on branch:**
- TASK-166 shipped at `8665fc3e`: `findAgentsDir` is now worktree-aware via `findMainWorktreeRoot` in `cli/lib/tasks/resolve.ts`. Every existing call site picks up the new behavior automatically.
- 11 tests in `cli/lib/tasks/resolve.test.ts` use real `git init` / `git worktree add` fixtures — reuse this pattern.

**Three concerns to ship in this task:**

1. **Migration command** `loaf migrate worktree-storage`:
   - Dry-run default; mutation behind explicit flag (implementer picks: `--apply` recommended)
   - Move every subdir + file under worktree-local `.agents/` into the main worktree's `.agents/`, preserving structure
   - Conflict policy: prefer most recently modified; `--force-from-worktree` / `--force-from-main` overrides
   - Write back-pointer file in worktree-local `.agents/` (implementer picks format; suggest `.moved-to` containing the main worktree absolute path)
   - Idempotent: re-run is no-op
   - When invoked from main checkout: decide between "no-op exit" vs "refuse with message" and document the choice
   - Round-trip test

2. **Refusal nudge** wired into the CLI's top-level command dispatcher:
   - Detect pre-A3 state: linked worktree where worktree-local `.agents/` contains content AND back-pointer is absent/stale
   - Refuse every loaf command except `migrate worktree-storage` (and `--help` / `--version`) with consistent messaging
   - Main checkout and single-worktree repos never trigger
   - Refusal message centralized (single source of truth — one exported constant)

3. **Observability knob** in `findMainWorktreeRoot`:
   - When `LOAF_DEBUG_RESOLVE=1` is set, write git stderr to `process.stderr` instead of suppressing
   - Single shared env var name constant referenced by both `resolve.ts` and the migrate command's help text
   - Surface in migrate command's `--help` as "use `LOAF_DEBUG_RESOLVE=1 loaf ...` to diagnose unexpected refusal nudge behavior"
